# PrestaShop Fixes

Small, tested Python and Node.js scripts that detect and repair real problems on **PrestaShop** stores. Negative and drifting stock, Webservice API data that will not sync, orders stuck on a stale state, order history gaps, duplicate references and combinations, broken friendly URLs, and multistore scoping mistakes.

Every fix is safe by default. The scripts start in a dry run mode that reports what they would do, so you can read the plan before anything writes.

## How the scripts work

The scripts talk to the **PrestaShop Webservice API**. The webservice key is sent as the HTTP Basic username (with a blank password) to the `/api/...` resources, read as JSON with `?output_format=JSON`. Node uses the built-in `fetch`; Python uses `requests`. The decision logic in every fix is a pure function with no I/O, so it is unit tested.

## Setup

Set the environment variables a fix needs. Enable the Webservice under Advanced Parameters, Webservice, and create a key.

```bash
export PRESTASHOP_URL="https://your-shop.example.com"
export PRESTASHOP_WS_KEY="your webservice key"
export DRY_RUN="true"   # start safe
```

Python needs `pip install requests pytest`. Node needs Node 18 or newer (the scripts use the built-in `fetch`, no packages).

## Running the tests

The decision logic in every fix is a pure function with no network calls, so the tests run anywhere.

```bash
# Python
pip install pytest
pytest

# Node
node --test
```

## A note on safety

These scripts can change orders, inventory, prices, and issue refunds. Always run with `DRY_RUN=true` first, read the output, and confirm it is correct before you let a script write. Test against a staging store when you can.
