# Catalog rule skipped by specific price

PrestaShop stores manual specific prices and catalog-rule-generated discounts in the same `specific_price` table. `Product::getPriceStatic()`, through `SpecificPrice::getSpecificPrice()`, always picks a manually-created row (`id_specific_price_rule = 0`) ahead of any row generated by a catalog rule, even when the rule is active and the product is in its scope. The rule's own generated row still exists, it is just never the one selected for that product, silently, with nothing flagged in the admin. This script scans your active catalog price rules, resolves each rule's target products, and reports every product where a manual override is blocking the rule.

**Full guide with diagrams:** https://www.allanninal.dev/prestashop/catalog-rule-skipped-by-specific-price/

## Run it

```bash
export PRESTASHOP_URL="https://your-shop.example.com"
export PRESTASHOP_WS_KEY="your webservice key"
export REPORT_PATH="skipped_catalog_rules.csv"
export DRY_RUN="true"

python catalog-rule-skipped-by-specific-price/python/catalog_rule_skip_report.py
node   catalog-rule-skipped-by-specific-price/node/catalog-rule-skip-report.js
```

`classify_skipped_product` / `classifySkippedProduct` is a pure function: a product is only flagged as skipped when it is in an active rule's target scope AND it carries a manual `specific_price` row (`id_specific_price_rule = 0`) whose date window covers now. That row is exactly what wins the priority order in `Product::getPriceStatic()`.

The script never auto-deletes or auto-modifies a merchant's manual override, that price was set on purpose. It only writes a CSV report of `{id_specific_price_rule, rule_name, id_product, product_reference, manual_price_or_reduction, id_specific_price, from, to}`. If a merchant opts in to a corrective action, the safe move is to either delete the specific, merchant-selected `specific_price` id, or shorten its `to` date so it expires and the catalog rule's own row becomes effective again. Start with `DRY_RUN=true` to review the report first.
